# Orbit

Orbit คือเว็บติดตามดาวเทียมไทยแบบ 3D ที่พัฒนาโดย Spaceth เพื่อให้ทุกคนเห็นได้ว่าดาวเทียมของไทยกำลังโคจรอยู่ตรงไหนเหนือโลก ผ่านภาพที่เข้าใจง่าย โต้ตอบได้ และไม่จำเป็นต้องใช้โปรแกรมติดตามดาวเทียมเฉพาะทาง

เว็บนี้ไม่ได้ “เดา” ตำแหน่งของดาวเทียมขึ้นมาเอง แต่ใช้ข้อมูลวงโคจรจริงที่เรียกว่า **TLE** แล้วนำมาคำนวณว่าดาวเทียมแต่ละดวงควรอยู่ที่ตำแหน่งใดในเวลาปัจจุบัน

## TLE คืออะไร

**TLE** ย่อมาจาก **Two-Line Element** เป็นรูปแบบข้อมูลขนาดเล็กที่ใช้อธิบายวงโคจรของดาวเทียมด้วยข้อมูลเพียงสองบรรทัด

ถ้ามองครั้งแรก TLE อาจดูเหมือนตัวเลขยาว ๆ ที่อ่านไม่รู้เรื่อง แต่ภายในนั้นมีข้อมูลสำคัญของวงโคจรอยู่ เช่น ความเอียงของวงโคจร ความเร็วในการโคจร รูปร่างของวงโคจร และเวลาที่ข้อมูลชุดนั้นอ้างอิงถึง

อธิบายง่าย ๆ TLE คือ “ภาพ snapshot ล่าสุด” ของวงโคจรดาวเทียม จากข้อมูลชุดนี้ เราสามารถใช้แบบจำลองทางคณิตศาสตร์คำนวณต่อได้ว่า ดาวเทียมควรอยู่ตรงไหนในอีกไม่กี่นาทีข้างหน้า อีกหลายชั่วโมงข้างหน้า หรือย้อน��ลับไปดูตำแหน่งในอดีตก็ได้

Orbit ไม่ได้เก็บตำแหน่งดาวเทียมไว้เป็นจุดสำเร็จรูป แต่จะดาวน์โหลด TLE ล่าสุด แล้วคำนวณตำแหน่งของดาวเทียมแบบ real-time ภายใน browser

## เว็บรู้ได้อย่างไรว่าดาวเทียมอยู่ตรงไหน

หลังจาก Orbit ได้ข้อมูล TLE แล้ว ระบบจะใช้แบบจำลองวงโคจรที่เรียกว่า **SGP4** เพื่อคำนวณตำแหน่งของดาวเทียม

SGP4 เป็นแบบจำลองมาตรฐานที่ใช้กับข้อมูล TLE มาอย่างยาวนาน ให้คิดว่ามันคือเครื่องคิดเลขวงโคจรแบบดั้งเดิม ที่ยังถูกใช้กันอย่างแพร่หลายในการติดตามดาวเทียมจนถึงทุกวันนี้

โปรเจกต์นี้ใช้ **satellite.js** ซึ่งเป็น JavaScript library สำหรับอ่านข้อมูล TLE และคำนวณตำแหน่งดาวเทียมด้วย SGP4

ขั้นตอนโดยรวมจะประมาณนี้

1. เว็บดาวน์โหลดข้อมูล TLE ของดาวเทียมแต่ละดวง
2. satellite.js อ่านข้อมูล TLE แล้วแปลงเป็นแบบจำลองวงโคจร
3. SGP4 คำนวณตำแหน่งของดาวเทียม ณ เวลาปัจจุบัน
4. ระบบแปลงตำแหน่งนั้นออกมาเป็น latitude, longitude, altitude และตำแหน่ง 3D บนโลก
5. เว็บแสดงตำแหน่งนั้นเป็นจุดดาวเทียมและเส้นวงโคจรบนโลก 3D

ดังนั้น TLE คือข้อมูลตั้งต้น, SGP4 คือวิธีคำนวณ และ satellite.js คือเครื่องมือที่ทำให้ทั้งหมดนี้ใช้งานบนเว็บได้จริง

## ข้อมูล TLE มาจากไหน

Orbit ดึงข้อมูล TLE ผ่าน API จาก

**tle.ivanstanojevic.me**

อย่างไรก็ตาม เว็บไม่ได้เปิดให้ผู้ใช้ดึงข้อมูล TLE ของวัตถุอวกาศอะไรก็ได้ตามต้องการ Orbit ใช้ internal API เฉพาะกับดาวเทียมไทยที่อยู่ในรายการของโปรเจกต์เท่านั้น

ฝั่ง server จะมี route สำหรับดึงข้อมูล TLE ตาม NORAD ID เช่น THEOS-2, KnackSat-2, THEOS หรือ Thaicom ก่อนจะดึงข้อมูล ระบบจะตรวจสอบก่อนว่า NORAD ID นั้นอยู่ใน allowed list หรือไม่

เหตุผลคือเพื่อไม่ให้ API ของเว็บกลายเป็น open proxy ที่ใครก็ใช้ดึงข้อมูลอะไรก็ได้ และยังช่วยให้เราควบคุมรายการดาวเทียมได้ง่ายขึ้นด้วย

ข้อมูล TLE จะถูก cache ไว้ประมาณ 1 ชั่วโมง เพราะวงโคจรดาวเทียมไม่ได้เปลี่ยนทุกวินาทีเหมือนราคาหุ้น การ cache จึงช่วยให้เว็บโหลดเร็วขึ้น และลดการเรียก API ที่ไม่จำเป็น

## TLE สำรอง (Fallback)

หาก API ของ TLE สดใช้งานไม่ได้ Orbit จะใช้ข้อมูล TLE สำรองที่ฝังไว้ในโปรเจกต์ที่ `src/data/tle-fallback.json`

ระบบ fallback ทำงานสองชั้น

1. **ฝั่ง server** — route `/api/tle/[id]` จะลองดึงจาก API สดก่อน หากล้มเหลวจะส่งคืน TLE สำรองของดาวเทียมนั้นแทน
2. **ฝั่ง browser** — หากเรียก API route ไม่สำเร็จ client จะใช้ข้อมูลสำรองชุดเดียวกัน เพื่อให้ยังติดตามดาวเทียมได้แม้ API ล่มหรือออฟไลน์

โลก 3D และรายการดาวเทียมยังทำงานได้ด้วยข้อมูลสำรอง ข้อความ error พร้อมปุ่ม **ลองอีกครั้ง** จะแสดงเฉพาะเมื่อทั้ง API สดและข้อมูลสำรองใช้ไม่ได้

อัปเดตไฟล์สำรองจาก API เดิม (แนะนำประมาณสัปดาห์ละครั้ง)

```bash
npm run update-tle-fallback
```

คำสั่งนี้จะดึง TLE ล่าสุดของดาวเทียมทุกดวงในรายการโปรเจกต์ แล้วเขียนทับ `src/data/tle-fallback.json`

## 3D View ใช้อะไรขับเคลื่อน

โลก 3D และการแสดงผลดาวเทียมของ Orbit สร้างขึ้นด้วย **Three.js** ผ่าน **React Three Fiber**

Three.js คือ library สำหรับสร้างกราฟิก 3D บนเว็บ ส่วน React Three Fiber ช่วยให้การใช้ Three.js ในโปรเจกต์ React ทำได้ง่ายและเป็นระบบมากขึ้น

ใน Orbit เครื่องมือเหล่านี้ถูกใช้ในการ render โลก 3D จุดดาวเทียม เส้นวงโคจร และการเคลื่อนกล้อง

โลกในฉากไม่ได้เป็นแค่ทรงกลมเรียบ ๆ แต่ใช้ข้อมูลแผ่นดินจาก Natural Earth นำมาแปลงเป็น texture แล้วหุ้มลงบน globe ทำให้ได้แผนที่แผ่นดินและมหาสมุทรที่ดูสะอาด เรียบง่าย และเข้ากับภาพรวมของเว็บ

จุดดาวเทียมถูกออกแบบให้มองเห็นได้เสมอ ไม่ว่าจะ zoom เข้าใกล้หรือถอยออกมาไกลแค่ไหน ขนาดของ marker จะถูกคำนวณใหม่ให้คงที่บนหน้าจอ แทนที่จะเล็กจนหายไปเมื่อ zoom out หรือใหญ่เกินไปเมื่อ zoom in

## Orbit ทำอะไรได้บ้าง

Orbit สามารถแสดงดาวเทียมไทยหลายดวงพร้อมกันบนโลก 3D ผู้ใช้สามารถเลือกดาวเทียมแต่ละดวงเพื่อดูรายละเอียดเพิ่มเติมได้ เช่น ตำแหน่ง ความสูง ความเร็ว ข้อมูลภารกิจ วันปล่อย และหน่วยงานผู้ดำเนินงาน

ผู้ใช้ยังสามารถซ่อนหรือแสดงดาวเทียมแต่ละดวงได้ตามต้องการ เหมาะเวลาที่อยากโฟกัสเฉพาะดาวเทียมบางดวง หรือเวลาที่หน้าจอเริ่มดูเหมือนมีใครทำเส้นบะหมี่ตกใส่โลก

เมื่อเลือกดาวเทียม กล้องจะเคลื่อนไปโฟกัสที่ดาวเทียมดวงนั้นอย่างนุ่มนวล และเมื่อยกเลิกการเลือก กล้องจะกลับมายังมุมมองโลกปกติ

Orbit ยังรองรับทั้ง Light Mode และ Dark Mode โดย interface และฉาก 3D ใช้ theme system เดียวกัน ทำให้ UI และ globe มีหน้าตาที่สอดคล้องกันทั้งเว็บ

รายการนี้สามารถอัปเดตได้จากไฟล์ข้อมูลดาวเทียมหลัก เมื่อเพิ่ม NORAD ID และ metadata พื้นฐานของดาวเทียมเข้าไปแล้ว ทั้ง API และ UI จะทำงานตามข้อมูลใหม่นั้นโดยอัตโนมัติ

## Tech Stack

Orbit พัฒนาด้วย **Next.js**, **React**, **TypeScript**, **Three.js**, **React Three Fiber**, **satellite.js** และ **Tailwind CSS`

สำหรับการคำนวณวงโคจร Orbit ใช้ satellite.js เพื่ออ่านข้อมูล TLE และคำนวณตำแหน่งดาวเทียมด้วย SGP4

สำหรับมุมมอง 3D ใช้ Three.js และ React Three Fiber

สำหรับการดึงข้อมูล ใช้ Next.js API Route เพื่อ fetch และ cache ข้อมูล TLE

สำหรับ styling ใช้ Tailwind CSS ร่วมกับ shared theme tokens สำหรับ Light Mode และ Dark Mode

## Deployment

Orbit เป็นโปรเจกต์ Next.js จึงสามารถรันบนเครื่องได้ด้วยคำสั่งมาตรฐาน

```bash
npm install
npm run dev
```

สำหรับ production

```bash
npm run build
npm run start
```

สรุปสั้น ๆ Orbit คือเว็บขนาดเล็กที่เชื่อมข้อมูลวงโคจรดิบ ๆ เข้ากับโลก 3D ที่มองเห็นและเล่นได้จริง มันเอาสิ่งที่ปกติซ่อนอยู่ในไฟล์ TLE แห้ง ๆ มาแปลงเป็น globe แบบ interactive เพื่อให้ทุกคนเห็นได้ว่า ดาวเทียมไทยกำลังบินอยู่ตรงไหนเหนือโลก
